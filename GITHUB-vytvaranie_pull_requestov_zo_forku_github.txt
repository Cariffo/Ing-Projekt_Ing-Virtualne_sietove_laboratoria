Vytváranie Pull Requestov do GitHub hlavného repozitára z forkov

-forkneme si nejaký repozitár od nejakého užívateľa

(Windows)
-otvoríme GitHub Desktop aplikáciu
-Naklonujeme si do ľubovoľného adresára forknutý repozitár (POZOR! klonujeme iba repozitár z nášho účtu, nie z účtu maintainera)
-upravíme obsah forknutého repozitára (pridáme/upravíme/vymažeme súbory)
-GitHub Desktop aplikácia zobrazí zmeny
-Vyplníme položky vpravo dole (TODO - ktoré) a klikneme na zelené tlačidlo "" (TODO - konkrétny text)
-Potom zmeny "commitneme" a "pushneme" do online GitHub (forknutého) repozitára

-Maintainer potom:
  -porovná svoj repozitár s forknutým repozitárom vytvorením Pull Requestu
  -zistí, aké zmeny sa vykonali
  -buď prijíme (zmerdžuje pull request) zmeny z forknutého repozitára alebo odmietne zmeny (zavrie pull request)

-zmeny sa v online repozitári maintainera prejavia okamžite po zmerdžovaní.
-aby sa prejavili aj lokálne, treba vykonať "git pull"
