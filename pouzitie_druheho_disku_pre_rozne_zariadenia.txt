TROUBLESHOOTING


-Ak sa server spustil, overime ci sa particia pripojila prikazom
  lsblk

-Ak zadame nepravne udaje do "/etc/fstab" suboru, hrozi, ze sa server nespusti a znefunkcni sa aj SSH pristup. V mojom pripade som pouzil nespravny prikaz na vygenerovanie UUID pre particiu ("uuidgen" namiesto "blkid"). Vtedy je potrebne fyzicky restartovat server, vojst do GRUB zavadzaca (dlhsie podrzime "Shift" pocas zavadzania systemu - ked sa spusta BIOS).

-Ak to nepojde, treba restartovat server znova a nabootovat z Live USB s napr. Ubuntu Desktop. V nom si otvorime terminal, pripojime systemovu particiu a vygenerujeme spravne UUID pre particiu /dev/sdb1 (vid "blkid" prikaz nizsie).

    at the grub prompt, hit a to append options
    add init=/bin/bash to the end of the kernel command line and press enter

The system will boot to a prompt like 'bash-3.2#' enter the following commands at the prompt

    mount -o remount,rw /
alebo
    mount -o remount,rw / --target /

Vygenerujeme UUID pre particiu /dev/sdb1, ale tentokrat spravnym sposobom.

    blkid -s PARTUUID -o value /dev/sdb1

UUID si ulozime (napr. odfotime).
Upravime fstab subor podla spravnej UUID hodnoty.

    vim.tiny /etc/fstab

Restartujeme

    reboot -f
