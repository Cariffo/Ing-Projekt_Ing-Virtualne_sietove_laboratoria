**********
   GNS3
**********
Výhody
-Multiplatformový - klientská aplikacia existuje pre Windows/Linux/Mac
-Možnosť nasadiť GNS3 aj ako vzdialený server
-Schopnosť skonvertovať Cisco smerovač z BIN do IMAGE formátu (uzitovne pre Dynamips emulaciu)
-Lepšia škálovateľnosť oproti EVE-ng - moznost vytvorit GNS3 cluster
-Podpora viacerych pouzivatelov
-Import pouzivatelskych uctov na vzdialeny GNS3 server z LDAP

Nevýhody
-Nutnost instalacie samostatnej klientskej aplikacie
-Zložitejší na konfiguráciu
-Zložitejšie vytváranie šablón pre zariadenia - nutnosť manuálne pridať každé zariadenie; pri GNS3 clustri treba každé zariadenie pridať samostatne na každý server v clustri

**********
  EVE-ng
**********
Výhody
-Web rozhranie namiesto samostatnej klientskej aplikácie v GNS3
-Podpora viacerých používateľov
-Jednoduchšie nasadenie na server v porovnaní s GNS3 (nainštalujeme EVE-ng server, zabezpečíme ho, pridáme doň zariadenia a môžeme vytvárať projekty)
-Vyššia bezpečnosť: používateľ sa pred použitím web rozhrania musí autentifikovať heslom.

Nevýhody
-Nutnosť vypnutia zariadenia, keď je potrebné pridať/ostrániť prepojenie s iným zariadením.
-Nevie skonvertovať (dekomprimovať) Cisco smerovač z BIN do IMAGE formátu.
-Všetci používatelia sú typu “Administrator”- ktokoľvek zaregistrovaný môže pridávať/upravovať/mazať projekty/adresáre/používateľov
-Web server (Apache) nie je chránený modulmi “modsecurity” (ochrana napr. Proti SQL Injection) a “modevasive” (ochrana proti DoS a DDoS útokom)
-EVE-ng sa nedá škálovať naprieč viacerými servermi t.j. nevieme urobiť EVE-ng cluster tak, ako to vieme urobiť v GNS3
